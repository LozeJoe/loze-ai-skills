## 方法论：需求信号反推法（来自 cheat-on-money）

**核心前提**: 没有人会把真正赚钱的方法发到「教你赚钱」的帖子里——那些是卖课/钓粉。

### 信源分级

| 等级 | 信源 | 作用 |
|------|------|------|
| ✅ 一级 | 招聘数据、机构报告、平台官方公告 | 主要证据 |
| 🛰 雷达 | GitHub 真人项目（1c7/chinese-independent-developer, loonggg/DevMoneySharing） | 发现方向 |
| ⚠️ 二级 | 中立媒体报道、真实用户吐槽 | 佐证 |
| ❌ 不作依据 | 「教你AI赚钱」「月入过万」类帖子 | 污染信源 |

### 四步推理

1. 采集信号（招聘数据/行业报告/GitHub真实项目）
2. 推理：谁有真需求 → 个人能否供给 → 买家够得着吗
3. 交叉验证（至少2个独立信号指向同一结论）
4. 落到可验证方案

## 反诈 Rubric（每个机会必过）

### A. 硬红线（命中=直接淘汰）

| # | 红线 |
|---|------|
| A1 | 入场付费（交押金/培训费/会员费） |
| A2 | 收益承诺（"日入过千""稳赚不赔"） |
| A3 | 拉人头返佣（传销结构） |
| A4 | 资金/账户经手（刷单/跑分/出租银行卡—涉嫌违法） |
| A5 | 卖「搞钱课」套娃 |
| A6 | 违法/灰产（代写毕设、伪造内容、爬取倒卖个人信息） |

### B. 存疑信号（需额外验证）

B1 无第三方真实收款证据 / B2 报酬远高于行情 / B3 绕开平台担保 / B4 催促名额有限 / B5 试岗任务 / B6 雇主查无此人

### 验证动作

1. 搜负面: `<平台名> 骗局`
2. 查实体: 备案/公司信息
3. 找真实反馈: 小红书/知乎/黑猫投诉
4. 验证收款: 小成本先试1单确认钱能到账

## 接单交付 SOP

```
买家私聊Loze → Loze问清需求 → Loze发我微信 → 我写代码 → Loze交付
```

### Loze 职责

- 闲鱼接单、沟通需求、报价、收款、交付
- 沟通格式发给我的消息：
```
有新单子：
类型：Python脚本/Vue前端/Java/其他
需求：xxx
预算：xxx
截止：xxx
```

### Hermes 职责

- 收到需求后 3-5 分钟评估可行性 + 报价建议
- 写完整可运行代码 + 使用说明
- 简单脚本 1h 内交付，复杂项目当天

### 报价参考

| 类型 | 起步价 | 典型范围 |
|------|--------|----------|
| 简单脚本（<50行） | ¥50 | 50-100 |
| 中等脚本 | ¥100 | 100-200 |
| 复杂项目 | ¥200 | 200-500+ |
| AI工具安装 | ¥30 | 30-80 |
| 网页修改 | ¥100 | 100-300 |
| 整页/组件 | ¥200 | 200-800 |

报价原则：比市场价低 20-30%（学生身份 + AI 效率优势）

### 交易安全

- 必须走闲鱼担保交易，不私下转账
- 不先交付完整代码再等付款
- 复杂项目分阶段交付（先核心功能→确认→继续→最终交付）

> 完整 SOP 见 `references/xianyu-sop.md`
