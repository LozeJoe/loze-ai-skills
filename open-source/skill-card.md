## Description: <br>
Find, evaluate, self-host, maintain, and publish open source projects with due diligence scoring, contributor workflows, and release governance. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[ivangdavila](https://clawhub.ai/user/ivangdavila) <br>

### License/Terms of Use: <br>


## Use Case: <br>
Developers, maintainers, and technical teams use this skill to compare open source projects, evaluate self-hosting readiness, structure maintainer operations, and prepare releases with practical governance checks. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Optional local memory can preserve open-source preferences, project context, and decision history. <br>
Mitigation: Use memory only when persistence is intended, and avoid storing confidential business, security, or licensing details unless they should be reused. <br>
Risk: Open source recommendations can become stale as maintainer health, release cadence, security posture, or licensing signals change. <br>
Mitigation: Verify current project evidence before adoption, contribution, self-hosting, or publication decisions. <br>
Risk: Self-hosting guidance can underestimate operational ownership if backups, upgrades, observability, and incident response are not assigned. <br>
Mitigation: Require explicit ownership, tested restore procedures, rollback rules, monitoring, and security responsibilities before recommending production self-hosting. <br>
